# Анализ: AI-рекомендательная торговая система для криптовалют

**Дата:** 2026-02-15
**Статус:** Анализ завершён, ожидает формирования плана реализации

---

## 1. Постановка задачи

Построить AI-powered рекомендательную систему для торговли криптовалютами на ByBit, которая:
- Агрегирует данные из множества источников (рынок, on-chain, sentiment, предсказательные рынки, новости)
- Анализирует их с помощью AI/ML
- Генерирует торговые сигналы с оценкой уверенности и риска
- Доставляет через Telegram и консольный дашборд
- На первом этапе — только рекомендации, без автоматического исполнения
- В перспективе — автоматическая торговля с плечами

### Компоненты

#### Уровень 1: Сбор данных (Data Layer)
- **Рыночные данные** — ByBit API через ccxt (цены, объёмы, стаканы, свечи)
- **On-chain данные** — whale tracking, крупные транзакции
- **Социальный сентимент** — Twitter/X, Reddit, Telegram группы
- **Предсказательные рынки** — Polymarket API
- **Новости** — RSS, crypto news aggregators

#### Уровень 2: Анализ (Analysis Layer)
- **Технический анализ** — индикаторы, паттерны, уровни поддержки/сопротивления
- **Фундаментальный анализ** — метрики проектов, токеномика
- **Сентимент-анализ** — LLM-powered анализ настроений рынка
- **On-chain анализ** — движения китов, потоки на/с бирж
- **Рыночный контекст** — данные Polymarket, макроэкономика
- **Скринер новых монет** — листинги, launchpads, IDO

#### Уровень 3: Генерация сигналов (Signal Layer)
- ML/AI модели для предсказания движений
- Мультифакторная скоринговая система
- Risk scoring для каждого сигнала
- Фильтрация ложных сигналов

#### Уровень 4: Доставка и UI (Delivery Layer)
- Telegram bot — сигналы и алерты
- n8n — автоматизация workflow
- Консольный дашборд (TUI) — обзор рынка
- Логирование и аналитика сделок

#### Уровень 5: Обучение и бэктестинг
- Исторические данные
- Бэктестинг стратегий
- Paper trading (демо-режим)
- Трекинг реальных результатов

## 4. Реалистичность целей: доходность

### Данные исследования

| Уровень | Месячная доходность | Годовая | Реалистичность |
|---------|-------------------|---------|----------------|
| Консервативный | 1–5% | 12–60% | Наиболее реалистичный |
| Умеренный | 5–12% | 60–85% | Возможно в бычьем рынке |
| Агрессивный | 15–50% | 100%+ | Неустойчивый, маркетинг |

### Факты
- Профессиональные крипто-фонды: 15–30% **в год**
- DCA-боты на 3Commas: в среднем ~18.7% годовых
- Лучший задокументированный результат: 12.8% за 30 дней (DCA на BTC)
- На малых счетах комиссии могут съедать до 50% прибыли

### Рекомендация
Начальная цель: **3–5% в месяц** (36–80% годовых). Это уже значительно выше рынка и при этом достижимо. Цель 10%/мес можно рассматривать как амбициозный ориентир, но не как ожидание.

## 5. Критический анализ: комиссии при бюджете $500–2000

При торговом капитале $1000 на ByBit:

| Тип ордера | Комиссия | Стоимость сделки (вход+выход) |
|------------|----------|------------------------------|
| Taker (рыночный) | 0.1% | ~$2.00 |
| Maker (лимитный) | 0.02% | ~$0.40 |

При интрадей-торговле (5 сделок/день × 30 дней = 150 сделок):
- Taker: **$300/мес** (30% капитала!)
- Maker: **$60/мес** (6% капитала)

### Вывод
С бюджетом $500–2000 интрадей на рыночных ордерах **нерентабелен**. Рекомендуемая стратегия:
1. Фокус на **среднесрок** (удержание 3–14 дней) — меньше сделок, меньше комиссий
2. Использование **лимитных ордеров** (maker fee в 5x дешевле)
3. Интрадей — только на самых сильных сигналах
4. Тщательный отбор точек входа, а не частая торговля

## 7. Оценка имеющейся инфраструктуры

### OpenClaw
- Мощный open-source AI-агент (бывший Clawdbot/Moltbot)
- 150k+ GitHub stars, поддержка 50+ сервисов
- Активно используется для: Polymarket, research, on-chain анализа
- **РИСК:** Cisco выявила уязвимости (data exfiltration, prompt injection)
- **ЗАПРЕТ:** НЕ давать доступ к кошелькам и API-ключам с правами на торговлю

### n8n
- 193+ готовых крипто-workflow
- Примеры: AI-анализ с GPT-4o, интеграции с биржами, Telegram-алерты
- Подходит для автоматизации pipeline сбора данных
- Снижает объём кода (визуальные workflow)

### ByBit API
- Зрелый API v5
- Rate limits: 600 запросов/5 секунд (IP), WebSocket до 500 соединений/5 мин
- Python SDK: pybit (официальный), ccxt (универсальный)
- **Testnet** для безопасного тестирования
- Demo trading account для paper trading

## 9. Выявленные риски

### Критические
1. **Нереалистичные ожидания доходности** — может привести к увеличению рисков ради цели
2. **Безопасность API ключей** — утечка = потеря всех средств
3. **OpenClaw security** — задокументированные уязвимости, не давать доступ к реальным средствам

### Высокие
4. **Overfitting ML моделей** — отличные бэктесты, провальная реальная торговля
5. **Качество sentiment данных** — боты, манипуляции, FUD/FOMO на Twitter
6. **Twitter/X API стоимость** — базовый план ~$100/мес, нужны альтернативы (LunarCrush, Santiment)

### Средние
7. **Торговые издержки** — на малых счетах комиссии съедают прибыль
8. **Психология новичка** — эмоциональные решения вопреки сигналам системы
9. **Регуляторные риски** — зависит от юрисдикции
10. **API downtime** — потеря сигналов в критические моменты

### Меры митигации
- Обязательный этап **paper trading** (минимум 1–2 месяца)
- API-ключи **только для чтения** на начальном этапе
- **Лимиты потерь** (stop-loss на уровне системы)
- Диверсификация источников данных
- Регулярный аудит безопасности
